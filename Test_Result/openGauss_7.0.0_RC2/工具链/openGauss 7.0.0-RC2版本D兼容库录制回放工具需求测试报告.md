# 1 概述

本特性对D兼容性库的sql进行录制回放，分别在linux、windows、docker等环境执行sql、并且在linux和windows上使用tcpdump录制sql，然后进行sql解析和回放。提前录制真实业务场景的流量之后在上线之前用新版本的openGauss进行压测，可以最大程度拦截隐藏的，可能在现网爆发的问题，针对该需求出具本测试报告。

# 3 版本概要测试结论、关键风险和规避措施

## 3.1 测试结论总结

需求共计执行23个用例，主要覆盖了录制回放工具的参数配置及基础功能、在不同环境上对录制的sql进行采集、解析并回放到openGauss端D兼容性数据库、对D兼容性库不同版本兼容性的测试、录制回放工具对D兼容性库的性能影响等测试场景，共发现4个问题单，整体质量一般。

| 测试活动                                             | 活动评价                                                     |
| ---------------------------------------------------- | ------------------------------------------------------------ |
| 参数合法性故障测试                                   | 将各参数修改为非法参数，然后启动录制回放工具，报错参数不合法，符合预期。 |
| 录制解析功能测试                                     | 通过tcpdump方式（linux通过执行jar包完成录制，windows则直接使用自带的tcpdump.exe）录制指定ip和端口获取过滤后的网络文件，使用parse对网络文件进行解析得到记录sql语句的json字符串并放在文件和数据库中。 |
| 对涉及资源检测的参数功能进行测试                     | 录制过程中可以检查linux环境的cpu、内存、磁盘这几种系统资源，若过程中占用超过参数限定的百分比则及时报错提示资源超出范围并停止录制过程 |
| 录制过程中对远端发送文件参数功能进行测试             | linux环境录制过程中可以向远端发送包含sql的文件，保证本地磁盘不会占满 |
| 文件传输故障测试                                     | 远端机器磁盘空间调整为100%，再次在linux环境执行录制及传输文件功能，参数设置成功，录制过程中有报错传输文件失败，录制回放工具退出 |
| 对sql语句覆盖参数功能进行测试                        | 对于解析完成后应该存放在数据库中的sql语句，可以通过参数控制是否覆盖掉原有的数据 |
| 录制过程异常退出故障测试                             | linux环境同时启动录制回放工具和监视进程，使用kill杀掉tcpdump和录制回放主进程任意一个模拟故障场景时，监视工具都会将另一个kill掉之后退出（windows环境需要手动kill） |
| 兼容ipv6测试                                         | 将录制回放工具中涉及到的参数都换成ipv6，观察录制回放工具的功能可以正常运行 |
| 单向免密场景文件传输测试                             | 设A机器可以免密连至B，B机器不能免密连至A，在B机器上设置录制方式为tcpdump，其他参数正常，启动录制功能，然后导入数据，观察tcpdump指定的存放网络文件的文件夹有文件，回放过程中无报错。在A机器配置scp.sh的各参数，启动scp.sh，观察指定位置的tcpdump录制的网络文件被传到A机器的指定位置 |
| 不同连接方式录制sql测试                              | 使用不同的连接方式连接数据库，然后执行D兼容性sql并进行录制解析回放。发现命令行执行sql语句时，整个过程中均无报错，解析所得sql正确，可以回放到openGauss端D兼容库中，结果符合D兼容性特性。但是如果使用jdbc等远程连接，解析sql时出现问题。 |
| 录制回放工具支持设置多ip功能测试                     | 将通过其他录制方式的包含json的文件放在指定位置，设置sql.storage.mode=json，sql.file.path为json文件路径，sql.file.name为json文件名，sql.replay.database.ip设置多个ip并用逗号隔开，sql.replay.database.port设置多个端口并用逗号隔开，例如：<br/>sql.replay.database.ip=7.225.83.177,7.225.82.179<br/>sql.replay.database.port=11335,11335<br/>启动回放功能，观察数据库中有数据，和json保持一致，回放过程中无报错 |
| 录制回放工具回放功能测试                             | 使用串行回放和并行回放方式，分别在全量回放、只回放查询、N倍压测只回放查询场景下进行sql回放，回放过程中无异常，回放结果符合D兼容性特性。 |
| 不同数据类型录制回放功能测试                         | 对不同类型的sql语句进行从录制到回放的整体功能测试，录制回放工具全程无异常，录制回放工具结果符合D兼容性特性。 |
| pbe语句录制回放功能测试                              | 对pbe语句进行从录制到回放的整体功能测试。录制回放工具全程无异常，录制回放工具结果符合D兼容性特性。 |
| 表对象录制回放功能测试                               | 对表对象（包括视图、函数、存储过程）进行从录制到回放的整体功能测试，录制回放工具全程无异常，录制回放工具结果符合D兼容性特性。 |
| 使用sysbench模型导入数据，录制回放工具对性能影响测试 | 执行和sysbench导入数据相似的D兼容性sql。设置录制方式为tcpdump，其他参数正常，启动录制功能，观察tcpdump指定的存放网络文件的文件夹是否有文件，回放过程中是否有报错，记录性能，观察启动录制时和不启动录制时的性能损失是否小于5%。测试时执行sql原耗时962s，启动录制后耗时983s，性能损耗为2.1% |
| 录制回放工具tcpdump解析性能测试                      | 设置录制方式为tcpdump，其他参数正常，启动录制功能，然后执行大量和sysbench导入数据相似的D兼容性sql，观察tcpdump指定的存放网络文件的文件夹是否有文件，回放过程中是否有报错，使用解析工具进行解析对录制下来的sql进行解析, 记录解析前网络文件的大小和解析后json文件大小和解析完毕的时间。解析前网络文件大小为255MB，解析后json大小为176MB，40w条sql12s解析完毕，计算得解析性能14.6MB/s |

## 3.2 约束说明

该特性可以录制容器、windows、物理机、虚拟机上的D兼容性sql，可以在linux系统和windows上执行，并且支持在openGauss7.0.0-RC1以上版本的D兼容性库上回放D兼容性sql。
