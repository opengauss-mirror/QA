# 1 概述

录制回放工具支持录制openGauss服务的网络流量，并从中解析出sql语句，并按自定义方式到目标数据库进行回放。提前录制真实业务场景的流量之后在上线之前用新版本的openGauss进行压测，可以最大程度拦截隐藏的，可能在现网爆发的问题。在需要确认不同版本间openGauss的sql兼容性和单条sql执行性能是下降的场合也可以使用该工具。针对该需求出具本测试报告。

# 3 版本概要测试结论、关键风险和规避措施

## 3.1 测试结论总结

需求共计执行16个用例，主要覆盖了参数配置及基础功能、两种方式录制openGauss的sql语句的功能、对tcp方式录制的网络文件的解析、对采集到的sql语句的进行回放等场景，共发现3个问题单，2个缺陷单，1个资料单，整体质量良好。

| 测试活动                                                     | 活动评价                                                     |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| 参数合法性故障测试                                           | 将各参数修改为非法参数，然后启动录制回放工具，报错参数不合法，符合预期。 |
| 使用tcpdump或attach方式对openGauss低版本执行的sql语句进行采集与解析，获取到包含sql语句的json字符串并存入文件或openGauss高版本数据库中 | 使用tcpdump或attach方式对openGauss低版本执行的sql语句进行采集，对tcpdump录制的网络文件可以进行解析，两种方式均可以获取到sql语句，并以json字符串的形式存入文件或者openGauss高版本数据库中。 |
| 录制回放工具回放功能测试                                     | 使用串行回放和并行回放方式，分别在全量回放、只回放查询、N倍压测只回放查询场景下进行sql回放，回放过程中无异常，可以保证回放后数据与记录的sql一致，符合预期 |
| 不同数据类型录制回放功能测试                                 | 对含有不同数据类型的sql语句进行从录制到回放的整体功能测试，覆盖两种方式录制和不同场景回放，录制回放工具全程无异常，可以保证回放后数据与记录的sql一致，符合预期 |
| pbe语句录制回放功能测试                                      | 对pbe语句进行从录制到回放的整体功能测试，覆盖两种方式录制和不同场景回放，录制回放工具全程无异常，可以保证回放后数据与记录的sql一致，符合预期 |
| 表对象录制回放功能测试                                       | 对表对象（包括视图、函数、触发器、存储过程）进行从录制到回放的整体功能测试，覆盖两种方式录制和不同场景回放，录制回放工具全程无异常，可以保证回放后数据与记录的sql一致，符合预期 |
| blob类型二进制文件对象录制回放功能测试                       | 对blob类型二进制文件对象进行从录制到回放的整体功能测试，覆盖两种方式录制和不同场景回放，录制回放工具全程无异常，可以保证回放后数据与记录的sql一致，符合预期 |
| openGauss使用tpcc导入数据，录制回放工具对tpcc性能影响测试    | 对openGauss执行tpcc测试，记录tpcc性能。设置录制方式为tcpdump，其他参数正常，启动录制功能，然后执行tpcc测试，观察tcpdump指定的存放网络文件的文件夹是否有文件，回放过程中是否有报错，记录tpcc性能，常态业务下性能影响小于5%，极限性能场景下，由于tcpdump占用网卡资源，可能导致性能下降 |
| 录制回放工具tcpdump解析性能测试                              | 设置录制方式为tcpdump，其他参数正常，启动录制功能，然后向openGauss中导入数据，观察tcpdump指定的存放网络文件的文件夹是否有文件，回放过程中是否有报错，使用解析工具进行解析对录制下来的sql进行解析,记录解析前网络文件的大小和解析后json文件大小和解析完毕的时间 |
