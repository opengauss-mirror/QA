# 1 概述

本特性对可以输入openGauss端A兼容性库的sql进行录制回放，使用tcpdump方式录制sql，然后进行sql解析和回放。提前录制真实业务场景的流量之后在上线之前用新版本的openGauss进行压测，可以最大程度拦截隐藏的，可能在现网爆发的问题，针对该需求出具本测试报告。

# 3 版本概要测试结论、关键风险和规避措施

## 3.1 测试结论总结

需求共计执行20个用例，主要覆盖了录制回放工具的参数配置及基础功能、对录制的sql进行采集、解析并回放到A兼容性数据库，共发现2个问题单，整体质量良好。

| 测试活动                                         | 活动评价                                                     |
| ------------------------------------------------ | ------------------------------------------------------------ |
| 参数合法性故障测试                               | 将各参数修改为非法参数，然后启动录制回放工具，报错参数不合法，符合预期。 |
| 录制解析功能测试                                 | 使用tcpdump方式录制指定ip和端口获取过滤后的网络文件，使用parse对网络文件进行解析得到记录sql语句的json字符串并放在文件和数据库中 |
| 对涉及资源检测的参数功能进行测试                 | 录制过程中可以检查cpu、内存、磁盘这几种系统资源，若过程中占用超过参数限定的百分比则及时报错提示资源超出范围并停止录制过程 |
| 录制过程中对远端发送文件参数功能进行测试         | 录制过程中可以向远端发送包含sql的文件，保证本地磁盘不会占满  |
| 文件传输故障测试                                 | 远端机器磁盘空间调整为100%，再次执行录制及传输文件功能，参数设置成功，录制过程中有报错传输文件失败，录制回放工具退出 |
| 对sql语句覆盖参数功能进行测试                    | 对于解析完成后应该存放在数据库中的sql语句，可以通过参数控制是否覆盖掉原有的数据 |
| 录制过程异常退出测试                             | 同时启动录制回放工具和监视进程，使用kill杀掉tcpdump和录制回放主进程任意一个模拟故障场景时，监视工具都会将另一个kill掉之后退出 |
| 兼容ipv6测试                                     | 将录制回放工具中涉及到的参数都换成ipv6，观察录制回放工具的功能是否正常运行 |
| 单向免密场景文件传输测试                         | 设A机器可以免密连至B，B机器不能免密连至A，在B机器上设置录制方式为tcpdump，其他参数正常，启动录制功能，然后导入数据，观察tcpdump指定的存放网络文件的文件夹有文件，回放过程中无报错。在A机器配置scp.sh的各参数，启动scp.sh，观察指定位置的tcpdump录制的网络文件被传到A机器的指定位置 |
| 不同连接方式录制sql测试                          | 使用不同的连接方式连接数据库，然后执行A兼容性sql并进行录制解析回放，整个过程中均无报错，解析所得sql正确，可以回放到openGauss端A兼容库中，结果符合A兼容性特性 |
| 录制回放工具支持设置多ip功能测试                 | 将通过其他录制方式的包含json的文件放在指定位置，设置sql.storage.mode=json，sql.file.path为json文件路径，sql.file.name为json文件名，sql.replay.database.ip设置多个ip并用逗号隔开，sql.replay.database.port设置多个端口并用逗号隔开，例如：<br/>sql.replay.database.ip=7.225.83.177,7.225.82.179<br/>sql.replay.database.port=11335,11335<br/>启动回放功能，观察数据库中有数据，和json保持一致，回放过程中无报错 |
| 录制回放工具回放功能测试                         | 使用串行回放和并行回放方式，分别在全量回放、只回放查询、N倍压测只回放查询场景下进行sql回放，回放过程中无异常，回放结果符合A兼容性特性 |
| 不同数据类型录制回放功能测试                     | 对不同类型的sql语句进行从录制到回放的整体功能测试，录制回放工具全程无异常，录制回放工具结果符合A兼容性特性 |
| pbe语句录制回放功能测试                          | 对pbe语句进行从录制到回放的整体功能测试，录制回放工具全程无异常，录制回放工具结果符合A兼容性特性 |
| 表对象录制回放功能测试                           | 对表对象（包括视图、函数、存储过程）进行从录制到回放的整体功能测试，录制回放工具全程无异常，录制回放工具结果符合A兼容性特性 |
| 使用tpcc导入数据，录制回放工具对tpcc性能影响测试 | 执行tpcc测试，记录tpcc性能。设置录制方式为tcpdump，其他参数正常，启动录制功能，然后执行tpcc测试，观察tcpdump指定的存放网络文件的文件夹是否有文件，回放过程中是否有报错，记录tpcc性能，观察性能损失是否小于5% |
| 录制回放工具tcpdump解析性能测试                  | 设置录制方式为tcpdump，其他参数正常，启动录制功能，然后执行大量和sysbench导入数据相似的A兼容性sql，观察tcpdump指定的存放网络文件的文件夹是否有文件，回放过程中是否有报错，使用解析工具进行解析对录制下来的sql进行解析, 记录解析前网络文件的大小和解析后json文件大小和解析完毕的时间 |

## 3.2 约束说明

该特性主要在linux系统上进行，支持在openGauss6.0.0以上版本的A兼容性库上回放A兼容性sql。
