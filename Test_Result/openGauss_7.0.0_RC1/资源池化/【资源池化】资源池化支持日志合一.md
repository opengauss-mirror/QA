# 1 概述

当前openGuass资源池化支持共享一份存储，对外呈现为一写多读的产品形态。在本架构下不在需要主备流式复制，各节点独立维护自己的xlog和控制文件，仅主节点会根据业务记录xlog。该方案与主备独立存储的部署方式相比，能够大幅度降低副本数量，节省存储空间。但是主备独立维护xlog的方式，仍存在一定程度的空间浪费，并提高了代码编写的复杂程度，不利于后续演进。
